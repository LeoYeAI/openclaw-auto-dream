## The Problem

Every AI agent forgets. Session ends, context gone. Files pile up. What was that decision from two weeks ago? Which workflow worked last time? Your agent has amnesia вҖ” functional, but forgetting everything the moment it sleeps.

**Auto-Dream fixes this.** Like the human brain consolidating memories during sleep, Auto-Dream runs periodic "dream cycles" that scan, extract, organize, score, link, and prune your agent's knowledge вҖ” automatically, safely, and intelligently.

### Why This Matters for MyClaw Users

Your MyClaw agent runs 24/7. It handles your projects, tracks your decisions, remembers your preferences, manages your workflows. Over weeks and months, it accumulates thousands of daily log entries. Without Auto-Dream, that knowledge sits in raw files вҖ” unsorted, unscored, disconnected. Your agent has the data but can't *think* about it.

Auto-Dream transforms your agent from a tool that executes commands into a partner that **understands context, learns from history, and connects the dots you didn't see**.

### Five Memory Layers

| Layer | Storage | What Goes Here |
|-------|---------|---------------|
| **Working** | LCM plugin (optional, detected at setup) | Real-time context compression & semantic recall |
| **Episodic** | `memory/episodes/*.md` | Project narratives, event timelines, story arcs |
| **Long-term** | `MEMORY.md` | Facts, decisions, people, milestones, strategy |
| **Procedural** | `memory/procedures.md` | Workflows, preferences, tool patterns, shortcuts |
| **Index** | `memory/index.json` | Metadata, importance scores, relations, health stats |

## Features

### рҹ§  Cognitive Dream Cycle

Runs automatically via cron (default: 4 AM daily). Three phases:

1. **Collect** вҖ” Scans unconsolidated daily logs (last 7 days), detects priority markers (`вҡ пёҸ PERMANENT`, `рҹ”Ҙ HIGH`, `рҹ“Ң PIN`, `<!-- important -->`), extracts decisions / people / facts / projects / lessons / procedures / open threads

2. **Consolidate** вҖ” Routes each insight to the correct memory layer, performs semantic deduplication, assigns unique IDs (`mem_NNN`), creates relation links between connected entries

3. **Evaluate** вҖ” Scores importance using `base_weight Г— recency Г— reference_boost / 8.0`, applies forgetting curves (>90 days + low importance вҶ’ archived, never deleted), calculates 5-metric health score, generates 1вҖ“3 non-obvious insights, writes dream report, sends notification

### рҹ“Ҡ Importance Scoring

Every entry gets a score on every dream cycle:

```
importance = (base_weight Г— recency_factor Г— reference_boost) / 8.0
```

- **Recency**: `max(0.1, 1.0 - days/180)` вҖ” gradual decay over 6 months
- **References**: `logвӮӮ(count + 1)` вҖ” logarithmic boost for frequently-referenced entries
- **Markers**: `рҹ”Ҙ HIGH` doubles base weight; `вҡ пёҸ PERMANENT` always scores 1.0

### рҹ“ү Intelligent Forgetting

Memories aren't deleted вҖ” they're gracefully archived:
- Entry must be >90 days unreferenced AND importance < 0.3
- Compressed to a one-line summary in `memory/archive.md`
- Original ID preserved for relation tracking
- `вҡ пёҸ PERMANENT` and `рҹ“Ң PIN` entries are immune

### рҹ•ёпёҸ Knowledge Graph

Entries are linked by semantic relations. The reachability metric measures graph connectivity:
- Union-find algorithm across all entry relations
- Detects isolated knowledge clusters
- Suggests cross-references to improve coherence

### рҹ©ә Health Score (5 Metrics)

```
health = (freshnessГ—0.25 + coverageГ—0.25 + coherenceГ—0.2 + efficiencyГ—0.15 + reachabilityГ—0.15) Г— 100
```

| Metric | What It Measures |
|--------|-----------------|
| **Freshness** | % of entries referenced in last 30 days |
| **Coverage** | % of knowledge categories updated in last 14 days |
| **Coherence** | % of entries with at least one relation link |
| **Efficiency** | How concise MEMORY.md stays (inversely proportional to line count) |
| **Reachability** | How well-connected the memory graph is |

## Safety

| Rule | Why |
|------|-----|
| Never delete daily logs | Immutable source of truth |
| Never remove `вҡ пёҸ PERMANENT` | User protection is absolute |
| Episodes are append-only | Narrative history preserved forever |
| Auto-backup on >30% change | Prevents accidental corruption |
| Index backup every cycle | Always recoverable |
| Secrets policy | Only consolidates secrets already present |
